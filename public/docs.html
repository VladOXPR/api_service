CUUB API

A complete reference for the CUUB platform API — manage users, stations, tickets, batteries, and rents. Every endpoint below includes the full request, required parameters, and an example response you can copy directly.

Base URL https://api.cuub.tech

Authentication

Most endpoints are reachable over HTTPS without a bearer token. The /token endpoint returns the Energo backend token, which is managed internally and refreshed automatically. The /pos/* endpoints (Rents lifecycle) are the exception — they require a shared secret; see the POS section.

Tip: Replace {id}, {station_id}, or {sticker_id} placeholders in the examples with your own values.

Tickets (Maintenance)

Maintenance tickets are stored with a station_id that must exist in stations. Each ticket carries an array of task labels — a legacy single string is accepted and treated as a one-element array.

Task labels (use for multi-select UIs):
High Batteries Low Batteries No Batteries Add Stack Broken Battery High Failure Rates Hardware Malfunction Unusually Offline Urgent Other Other
UI hint: style Urgent Other as highest-priority (red), and Other as lower-priority (yellow).
GET /tickets

List all tickets

Optionally filter by one or more task labels. Filtering uses PostgreSQL array overlap — tickets are included if their task array contains any of the filter values.

Query parameters
task
string[] optional, repeatable
Filter to tickets whose task array contains any of the provided labels.
Request
cURL
curl -G "https://api.cuub.tech/tickets" \
  --data-urlencode "task=Low Batteries" \
  --data-urlencode "task=Other"
Response
JSON
{
  "success": true,
  "data": [
    {
      "id": 1,
      "location_name": "Kizami Sushi",
      "station_id": "{station_id}",
      "latitude": 41.88,
      "longitude": -87.63,
      "task": ["Low Batteries", "Hardware Malfunction"],
      "description": "Optional notes"
    }
  ],
  "count": 1
}
GET /tickets/{id}

Get a ticket by ID

{id} is the numeric ticket primary key (not a station id).

Request
cURL
curl -X GET https://api.cuub.tech/tickets/{id}
POST /tickets

Create a new ticket

Body parameters
location_name
string required
Human-readable location name.
station_id
string required
Must match an existing stations.id.
latitude
number required
Between -90 and 90.
longitude
number required
Between -180 and 180.
task
string[] required
Non-empty array of labels from the task list above (duplicates deduplicated).
description
string optional
Free-form notes.
Request
cURL
curl -X POST https://api.cuub.tech/tickets \
  -H "Content-Type: application/json" \
  -d '{
    "location_name": "Kizami Sushi",
    "station_id": "{station_id}",
    "latitude": 41.88,
    "longitude": -87.63,
    "task": ["Low Batteries", "Hardware Malfunction"],
    "description": "Optional notes"
  }'

Request
cURL

  -H "Content-Type: application/json" \
  -d '{
    "location_name": "Updated location name",
    "task": ["Hardware Malfunction", "Urgent Other"],
    "description": "Updated notes"
  }'

POS (Rents lifecycle)

Record the rent lifecycle in POS-DB.rents: start a rent when a powerbank is taken, close it when it's returned, and list all rent rows. rent_id is auto-generated as R#####; stripe_pi is unique per rent.

Authentication: every /pos/* request must include the shared POS_API_TOKEN secret as Authorization: Bearer <token> or x-api-key: <token>.
Errors: 401 if no header is sent, 403 if the token is wrong, 503 if the server has no POS_API_TOKEN configured (fail-closed).
POST /pos/start_rent

Start a new rent

Records a new rent when the powerbank is taken. rent_id is generated server-side.

Body parameters
stripe_pi
string required
Unique Stripe Payment Intent id for the rent.
station_start
string required
Station id where the rent began.
battery_id
bigint optional
Numeric id of the rented battery.
start_time
ISO timestamp optional
Defaults to server now() if omitted.
Request
cURL
curl -X POST https://api.cuub.tech/pos/start_rent \
  -H "Authorization: Bearer $POS_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "battery_id": 1042,
    "stripe_pi": "pi_3OabCdEfGhIjKlMn0001",
    "station_start": "STATION001"
  }'
Response
JSON
{
  "success": true,
  "data": {
    "rent_id": "R00042",
    "battery_id": "1042",
    "stripe_pi": "pi_3OabCdEfGhIjKlMn0001",
    "start_time": "2026-05-02T18:00:00.000Z",
    "station_start": "STATION001",
    "end_time": null,
    "station_end": null
  },
  "message": "Rent started successfully"
}
Error responses
• 400 Missing/invalid field (e.g. non-integer battery_id, blank stripe_pi).
• 409 Duplicate stripe_pi — a rent already exists for that payment intent.
PATCH /pos/end_rent

End a rent (battery returned)

Closes the currently-open rent for the returned powerbank by writing end_time and station_end. The rent is identified by battery_id — the endpoint locates the single rent with that battery_id where end_time IS NULL.

Body parameters
battery_id
bigint required
Id of the returned powerbank. The open rent for this battery is closed.
station_end
string required
Station id where the battery was returned.
end_time
ISO timestamp optional
Defaults to current server time if omitted.
Request
cURL
curl -X PATCH https://api.cuub.tech/pos/end_rent \
  -H "Authorization: Bearer $POS_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "battery_id": 1042,
    "station_end": "STATION007"
  }'
Response
JSON
{
  "success": true,
  "data": {
    "rent_id": "R00042",
    "battery_id": "1042",
    "stripe_pi": "pi_3OabCdEfGhIjKlMn0001",
    "start_time": "2026-05-02T18:00:00.000Z",
    "station_start": "STATION001",
    "end_time": "2026-05-02T20:30:00.000Z",
    "station_end": "STATION007"
  },
  "message": "Rent ended successfully"
}
Error responses
• 400 Missing/invalid battery_id, blank station_end, or malformed end_time.
• 404 No rent exists for the supplied battery_id.
• 409 Every rent for that battery is already closed, or more than one open rent exists for the same battery (response includes the conflicting rent_ids).

Rents

Rent data comes from Stripe balance transactions, aggregated by date or by station. Date ranges use YYYY-MM-DD_YYYY-MM-DD (America/Chicago).

GET /rents/{station_id}/{startDate}_{endDate}

Rent data for a station within a date range

Request
cURL
curl -X GET https://api.cuub.tech/rents/STATION001/2026-01-01_2026-01-31
Response
JSON
{
  "success": true,
  "data": {
    "station_id": "STATION001",
    "dateRange": "2026-01-01_2026-01-31",
    "totalAmount": 45.50,
    "totalRents": 15
  }
}
GET /rents/{dateRange}

Rents by date range (with previous-month comparison)

Per-day rent count and net sum from Stripe balance transactions. Includes previous-month comparison (ppositive, pnegative, prents, pmoney). Filtered by REVENUE_TYPES.

Request
cURL
curl -X GET https://api.cuub.tech/rents/2025-02-01_2025-02-08
Response
JSON
{
  "success": true,
  "range": "Feb 1, 2025 – Feb 8, 2025",
  "positive": 426,
  "negative": -57,
  "ppositive": 380,
  "pnegative": -42,
  "data": [
    { "date": "Feb 1, 2025", "rents": 5, "money": "$15" },
    { "date": "Feb 2, 2025", "rents": 6, "money": "$18" }
  ]
}
GET /rents/{dateRange}/all

Rents by date range — all stations

Net revenue per station: money = positive − negative. Only stations with at least one charge in the period and existing in the DB.

Request
cURL
curl -X GET https://api.cuub.tech/rents/2025-02-01_2025-02-08/all
Response
JSON
{
  "success": true,
  "range": "Feb 1, 2025 – Feb 8, 2025",
  "data": [
    { "station_title": "Annoyance Theater", "stripe_id": "cus_TeYJ78uzshxpCu", "money": 120.5 },
    { "station_title": "Kizami Sushi", "stripe_id": "cus_TvTm9txqLOIDGI", "money": 85.25 }
  ]
}
GET /rents/recent

Recent rents (limit only)

Aggregated rents for the most recent N balance transactions, with no date filter.

Query parameters
limit
number optional
Number of balance transactions to include. Default 10, max 100.
Request
cURL
curl "https://api.cuub.tech/rents/recent?limit=10"
